(1)	題目發想
題目發想於家裡附近吃很多年的攤販，大多數都是用紙本記錄點餐或是靠店員超強記憶。紙本紀錄的話，每點一餐就會用一張紙，浪費紙資源，若是用護貝的就會整張都髒髒的很醜，有時候糊到難以辨認；有些店員的記憶時好時壞，常常會拿到餐點發現跟點的不一樣或是送錯。我認為需要一個更加統一方便的點餐系統去避免以上的狀況。
為了協助店家更有效率地管理銷售流程，本專題設計了一套 「賣家用點餐系統」，提供直覺化的點餐操作介面、即時的銷售統計功能，以及簡單的商品管理介面，使店家能以最低成本建立電子化營業流程。
(2)	功能說明
系統主要分為三個分頁，各自負責不同功能模組：
1.點餐
•	顯示所有可販售商品，以按鈕（Button）方式呈現，顯示品項名稱、價格、庫存，若販售完畢會已灰色呈現且不可選取。商品數量如果過多提供滾動觀看功能。
 
•	點擊商品按鈕可選擇數量並自動加入訂單清單（不可超過庫存數量或填寫負數數量），若重複點選相同品項會自動合併成相同商品（也會檢查庫存數量及負數數量）。
  
•	提供修改、刪除已選商品數量功能
  
•	可檢視目前訂單內容（品項、數量、小計、總金額），送出後會自動紀錄至資料庫。 
 
2.商品設定頁面（Product / Setting）
 
•	可新增商品（包含名稱、非負價格、非負庫存量等資訊），提供修改、刪除功能。不能新增相同的商品名。
 
•	上方即時顯示目前所有商品資料（從資料庫讀取）。
3.訂單明細
 
•	分成「未完成訂單」以及「已完成訂單」兩區域
•	確認訂單後會顯示訂單（編號:訂單時間、各商品:名稱+數量+金額小記、動單總金額）於「未完成訂單」區域
•	若完成訂單可點擊訂單，會變換到「已完成訂單」區域
